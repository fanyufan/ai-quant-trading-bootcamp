复盘 & 策略管理 -- 使用说明

这个 Tab 是干嘛的

赚了是运气还是实力? 这个 Tab 用统计方法回答这个问题, 同时管理多策略生命周期。

三个子功能

  • Brinson 归因 -- 把超额收益拆成"配置效应 (赌赛道)" + "选股效应 (Alpha)" + "交互". 4 个数据源任选: CSV 真实交割单 / 模拟盘快照 / 实盘 (miniQMT 只读) / 演示数据. 顶部 "交割单 / 历史成交" 折叠面板可以单独看订单流水 + 成本水位 (bps).
  • Walk-Forward 过拟合检测 -- 滚动训练/评估窗口, 暴露策略真实 OOS 表现, 上实盘前的核心检测
  • 策略生命周期 -- 5 阶段管理 (Incubating/Paper/Probation/Production/Retired), KPI 驱动迁移

典型场景

场景 1 (周末复盘): 跑 Brinson, 看本周赚的钱主要来自赌赛道还是选对股; 选股效应弱 → 多因子模型要调整

场景 2 (上线前过拟合体检): 一个新策略想从 incubating 升 paper, 先在 Walk-Forward 里跑; OOS/IS ≥ 0.7 才放行, 否则压缩参数空间 / 换样本继续调

场景 3 (策略管理): Probation 策略连续 30 天 Sharpe < 0.5 → 自动 Retired; Paper 策略 Sharpe > 1.5 + 30 天稳定 → 升 Production

数据来源

  • Brinson 归因: 顶部"数据源"下拉支持三个选择
    • 模拟盘 (默认): 用 live_state.json 里的当前持仓 + 沪深300 当期成分股 (xtdata) + 申万一级行业 (xtdata) 算真实 Brinson. 区间默认 2026-04-01 至今, 可改
    • 实盘 (miniQMT 只读): 实盘账户当前持仓 + 同上的基准/行业字典
    • 演示数据: 一组固定示例数字 (组合 6.43%, 基准 0.72%), 与实盘无关, 用来对照公式
  • 申万一级字典: xtdata 拉取后缓存到 outputs/sw1_industry_map.json, TTL 7 天; 可点"刷新行业字典"立即更新
  • 策略库: outputs/strategy_registry.json

Brinson 实现说明

  • 基准: 沪深300 行业内等权 (该行业有几只成分股 / 300, 不按市值加权)
  • 持仓: 当前快照, 不做时段加权 (区间内换仓不影响, 适合短区间)
  • 境外标的 (e.g. 513100.SH 纳指 ETF): 归到 "境外/其他" bucket, 进入"配置效应" (基准无此行业)
  • 区间收益: end 收盘 / start 开盘 - 1, 来自 MySQL trade_stock_daily (回退 xtdata)

Walk-Forward 实现说明

  • 流程: 在每个时间窗口的训练期 (默认 120 天) 从候选参数里挑 Sharpe 最高的, 然后用这组参数在紧接的评估期 (默认 60 天) 跑 OOS, 滚动到下一段重复
  • 4 个内置策略: 双均线 (趋势) / 布林带反转 / RSI 反转 / 唐奇安通道突破 -- 覆盖经典三大风格
  • OOS/IS 比例: ≥ 0.7 相对鲁棒, 0.5~0.7 中度衰减, < 0.5 严重过拟合
  • 过拟合分数: (IS_Sharpe - OOS_Sharpe) / IS_Sharpe, < 30% 鲁棒 / 30~70% 中度 / > 70% 严重
  • 注意: 当 IS Sharpe ≤ 0 (策略训练期都不赚钱) 时 OOS/IS 与过拟合分数会被强制设为 0, 此时 verdict OK 是假阳性, 必须 IS > 0 + OOS/IS ≥ 0.7 才算真鲁棒
  • 典型经验: A 股个股 + 简单技术指标策略 (双参数级别) 几乎都会显示严重过拟合, 这是策略本身的局限. 真要找鲁棒策略需要换更复杂的模型 (多因子选股 / 风险平价 / 配对交易等)
