这个 Tab 是干嘛的

系统级运维面板, 一眼看出"哪个组件挂了"。还有标准 cron 时点参考。

健康检查 7 项

  • Python 版本 -- 运行环境
  • xtdata 行情 -- 是否能拉到 K 线/tick
  • DASHSCOPE_API_KEY -- Charles 投研对话能不能用
  • QMT_PATH -- 实盘下单路径 (默认 dry-run 不需要)
  • live_state.json -- 实盘 state 文件
  • strategy_registry.json -- 策略库
  • 历史晨报 -- 累积了多少份

定时任务

实盘工作台默认 3 个时点 (周一到周五):

  • 08:30 数据增量（根目录 .env 中 CASE_A_BOARD_DATA_PREP_DIR 下的 run_daily.py）
  • 09:30 开盘启动 dry-run 主循环
  • 14:55 收盘前 5 分钟停主循环

实现见 scheduler.py（APScheduler + cron）。 跟 web 分进程跑: python scheduler.py.

典型场景

场景 1 (排错): 投研对话不出回复 → 查"DASHSCOPE_API_KEY" 是不是 OK

场景 2 (上线前自检): 启动 cron 调度前先点"健康检查", 全 OK 再跑
