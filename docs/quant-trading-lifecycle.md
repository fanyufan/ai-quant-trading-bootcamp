# 量化投资完整流程

量化投资的完整流程通常可分为以下 7 大阶段，每个阶段环环相扣，且需要不断迭代优化。

本训练营的课程设计也围绕这 7 个阶段展开，从数据获取、因子挖掘、模型构建，到回测验证、风险管理、交易执行，再到实盘监控与迭代，形成完整闭环。

---

## 一、数据获取与处理（Data Acquisition & Processing）

这是量化策略的基石，数据质量直接决定策略上限。相关课程内容见 **Week 2：基础设施打造 + 数据获取与清洗**。

| 环节 | 具体内容 |
| --- | --- |
| **数据源** | 行情数据（价量、Tick）、基本面数据（财报、公告）、宏观数据、另类数据（舆情、供应链、卫星图像） |
| **数据清洗** | 处理缺失值、异常值、复权调整、停牌处理、ST/退市过滤 |
| **数据对齐** | 时间戳统一、截面数据与面板数据整合、多频率数据融合 |
| **数据存储** | 时序数据库（如 DolphinDB、ClickHouse）、本地文件（HDF5、Parquet） |

---

## 二、因子挖掘与特征工程（Factor Discovery & Feature Engineering）

将原始数据转化为具有预测能力的信号。相关课程内容见 **Week 5：缠论量化 + 网格与多因子**、**Week 6：机器学习因子挖掘 + 论文复现与策略进化**、**Week 11：投资晨会**。

| 环节 | 具体内容 |
| --- | --- |
| **Alpha 因子** | 动量、反转、价值、质量、波动率、流动性等经典因子；自定义量价/基本面组合因子 |
| **特征工程** | 标准化（截面 Z-Score、行业中性化）、降维（PCA）、非线性变换、时序特征（滑动窗口统计） |
| **因子检验** | IC 分析（信息系数）、分层回测、因子衰减分析、拥挤度检测 |
| **因子正交化** | 去除与常见风险因子（市值、行业）的相关性，提取纯净 Alpha |

---

## 三、模型构建（Model Construction）

将因子组合成可交易的预测信号。相关课程内容见 **Week 6：机器学习因子挖掘 + 论文复现与策略进化**、**Week 12：毕业路演（ML 概率策略）**。

| 模型类型 | 适用场景 |
| --- | --- |
| **线性模型** | OLS、Ridge、Lasso —— 可解释性强，适合中低频 |
| **树模型** | XGBoost、LightGBM、随机森林 —— 擅长捕捉非线性关系 |
| **深度学习** | LSTM、Transformer、TabNet —— 适合高维数据与时序模式挖掘 |
| **组合优化** | 均值-方差、风险平价、Black-Litterman —— 资产配置层面 |

关键步骤：样本内外划分、交叉验证、防止过拟合（正则化、Dropout、早停）。

---

## 四、回测验证（Backtesting）

在历史数据上模拟策略表现，评估其有效性。相关课程内容见 **Week 3：Backtrader 回测 + Talib 技术指标库**、**Week 5：缠论量化 + 网格与多因子**、**Week 7：QuantStats 绩效分析与报告**。

| 环节 | 具体内容 |
| --- | --- |
| **回测框架** | 事件驱动回测（如 Zipline、Backtrader、自研框架） |
| **交易成本** | 佣金、滑点、冲击成本、印花税、融资利率 |
| **调仓规则** | 再平衡频率（日频、周频、月频）、换仓时点、持仓数量 |
| **评价指标** | 年化收益、夏普比率、最大回撤、Calmar 比率、胜率、盈亏比 |
| **稳健性检验** | 参数敏感性分析、不同市场环境测试、蒙特卡洛模拟 |

---

## 五、风险管理（Risk Management）

量化策略的核心不是追求收益，而是控制风险。相关课程内容见 **Week 10：强化学习与风控体系 + 团队架构设计**、**Week 12：团队复盘与持续进化（Brinson 归因、Walk-Forward 过拟合检测）**。

| 层面 | 具体措施 |
| --- | --- |
| **事前风控** | 因子暴露限制（Barra 风险模型）、行业/市值中性约束、个股集中度上限 |
| **事中风控** | 实时 PnL 监控、止损线（硬止损/回撤止损）、流动性预警、异常波动熔断 |
| **事后风控** | 归因分析（Brinson 归因、因子归因）、策略失效检测、压力测试 |
| **组合层面** | 多策略低相关性配置、动态杠杆调整、尾部风险对冲（期权、期货） |

---

## 六、交易执行（Execution）

将模型信号转化为实际持仓，执行效率直接影响最终收益。相关课程内容见 **Week 9：Xtquant 实盘与 Agent 搭建实战**、**Week 11：实盘作战与 CEO 控制台**。

| 环节 | 具体内容 |
| --- | --- |
| **订单拆分** | TWAP（时间加权）、VWAP（成交量加权）、冰山订单、智能路由 |
| **冲击成本模型** | 根据市场深度和订单大小预估价格冲击 |
| **算法交易** | 自研执行算法或接入券商/第三方算法（如迅投、卡方、非凸） |
| **系统对接** | 通过 API（如 QMT、Ptrade、中泰 XTP、CTP）连接券商/期货柜台 |
| **实盘验证** | 先用小资金模拟盘/影子账户验证，再逐步放大 |

---

## 七、实盘监控与迭代（Live Monitoring & Iteration）

策略上线后并非一劳永逸，需要持续进化。相关课程内容见 **Week 7：QuantStats 绩效分析与报告**、**Week 11：投资晨会 + 实盘作战与 CEO 控制台**、**Week 12：团队复盘与持续进化 + 毕业路演**。

| 环节 | 具体内容 |
| --- | --- |
| **实时监控** | 策略盈亏、持仓偏离、因子表现、模型预测准确度、交易延迟 |
| **绩效归因** | 区分收益来源：Alpha 收益 vs 风格/运气收益 |
| **失效检测** | 滚动回测对比、因子 IC 衰减、策略拥挤度上升信号 |
| **策略迭代** | 定期重新训练模型、引入新数据源、因子挖掘、参数滚动优化 |
| **版本管理** | 策略版本控制、A/B 测试、灰度上线 |
